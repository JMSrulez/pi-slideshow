# Installation de pi-slideshow sur Raspberry Pi (Bullseye)

## Overview
Pi Slideshow is a simple photo slideshow program for the Raspberry Pi. This guide will help you install and run the application quickly and effectively.
To run on Raspberry PI 3B+ version process have been adjusted, so you can use cheap refurbished legacy rasperry pi.  
(next step will be english translation....)

0. **Pré‑requis**
- Raspberry Pi OS **Bullseye** (for Pi 3B+... better use latest version on RPi 4 or 5)
- Wi‑Fi setup (or ETH as you like).  
- SSH activated.
- update Hostname,
- expand file system
- switch off screen blanking (important for kiosk mode)

## Installation Steps

1. **Create dedicated user**  
Connected in pi (or another admin user):  
 ```bash
sudo adduser slideshow
 ```
Choose a password for slideshow.  
Other questions (Name, etc.) can be left blank (Enter).

Optional: add the user to the sudo group to give it administrative rights:
 ```bash
sudo usermod -aG sudo slideshow
```

2. **Update the Package List**  
Run the following command to ensure all your packages are up-to-date:
```bash
sudo apt update
sudo apt upgrade
```

3. **Install Docker on the Raspberry Pi**  
Install Docker with the official script:  
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
```

Then add the slideshow user to the docker group to be able to launch the containers without sudo:
   ```bash
sudo usermod -aG docker slideshow
   ```
Disconnect‑you /reconnect‑you (or restart) for the group to take effect.  
use the slideshow account as the kiosk account (or another...)  

Verify that Docker is working:
   ```bash
docker run hello-world
   ```
You should see a "Hello from Docker! ".

4. **Clone pi‑slideshow**
repository Logged in as a slideshow user:
  ```bash
cd ~
git clone https://github.com/JMSrulez/pi-slideshow.git
cd pi-slideshow
  ```

5. **Build and launch the container**
From the ~/pi-slideshow folder:
  ```bash
docker build -t pi-slideshow .
  ```
On a Raspberry Pi, construction can take several minutes (for example ~10 minutes, tested on a fresh install at 596s).
It depends mainly on speed of the SD card.

Then start pi-slideshow container
   ```bash
docker run -d \
  --name pi-slideshow \
  -p 5000:5000 \
  -v /home/slideshow:/home/slideshow \
  -e PI_SLIDESHOW_TITLE="Pi Slideshow - Écran 1" \
  -e HOME_FOLDER="/home/slideshow" \
  -e SHOWFILE="video_pi3_photos.mp4" \
  pi-slideshow
   ```

The application will be accessible at: http://<ip_of_pi>:5000 (for example http://192.168.1.42:5000).

6. **Example script to display the generated video in a loop**
Copy the example script to /usr/local/bin:
  ```bash
sudo cp scripts/mpv-kiosk.sh /usr/local/bin/mpv-kiosk.sh
  ```
Make the script executable:
  ```bash
sudo chmod +x /usr/local/bin/mpv-kiosk.sh
  ```
Check or adapt script variables:
```bash
sudo nano /usr/local/bin/mpv-kiosk.sh
```
Check the following lines at the beginning of the file:
  ```bash
VIDEO_PATH="${VIDEO_PATH:-/home/slideshow/video_pi3_photos.mp4}"
DISPLAY_VALUE="${DISPLAY_VALUE:-:0}"
  ```
VIDEO_PATH : full path of the video  
Default value : /home/slideshow/video_pi3_photos.mp4  

7. **Create a service to launch MPV at startup**
Copy the service to /etc/systemd/system From the project file:  
  ```bash
cd ~/pi-slideshow
sudo cp systemd/mpv-kiosk.service /etc/systemd/system/mpv-kiosk.service
  ```
Adapt the user and video path Edit service file:
  ```bash
sudo nano /etc/systemd/system/mpv-kiosk.service
  ```
Check and adapt these lines:
  ```bash
[Service]
User=slideshow
Group=slideshow
Environment=DISPLAY=:0
Environment=VIDEO_PATH=/home/slideshow/video_pi3_photos.mp4
ExecStart=/usr/local/bin/mpv-kiosk.sh
Restart=always
RestartSec=5
  ```
User / Group : the user who opens the graphics session (e.g. slideshow).

DISPLAY : must match the display of session X (usually:0).

VIDEO_PATH : must be the same as in the script, and point to the video generated by pi-slideshow.

Reload systemd and activate the service Run:
  ```bash
sudo systemctl daemon-reload
sudo systemctl enable --now mpv-kiosk.service
  ```
Check operation, Check service status:
  ```bash
systemctl status mpv-kiosk.service
  ```
The service must be “active (running)”.

If a valid video already exists in the VIDEO_PATH location, MPV should boot and loop it to the screen connected to the Raspberry Pi.  
In case of problem, consult the logs:
  ```bash
journalctl -u mpv-kiosk.service -e
  ```
Operation with pi-slideshow:  
- The pi-slideshow application (in Docker) writes the final video to the file configured in VIDEO_PATH (default: /home/slideshow/video_pi3_photos.mp4).  
- The mpv-kiosk.sh script waits for this file to exist and be non-zero in size, then loops MPV on it. When a new video is generated by pi-slideshow (atomatically replaced file). 
- MPV automatically restarts at the next playback at file/folder event or when restarting the service, and broadcasts the new version.
